# v3.1 运行前协议勘误：绝对 cycle 前缀审计

本勘误在任何 v3.1 模型、测试或数据处理运行**之前**提交。它只消除主协议中“prefix cutoff=`[0.35, 0.60]`”可能涉及目标相对完整进度的歧义；不根据结果改变任何 Gate A、B、C 的成功阈值、特征、数据集、随机种子或模型参数。

从本勘误开始，通用因果审计的两个固定切断点为目标实验的绝对 `center_cycle=3000` 和 `center_cycle=9000`。审计仅在切断点之后向特征注入固定扰动，并比较切断点及之前的预测、BOCPD、private-state 与连续进程输出。切断点不由目标最终长度、目标窗口数或相对完整进度派生；最大允许绝对差仍为 `1e-12`。

本勘误不改变下列已冻结项目：Gate A 的 1% 改善和 1.05 post-gate 比值；Gate B 的 BOCPD/片段/BIC 参数；Gate C 的 200 个共同到达窗口、0.50 NRMSE 与 20% OOD 不确定性提升要求；以及全部禁止输入和禁止操作。后续正式运行只能使用本勘误。
