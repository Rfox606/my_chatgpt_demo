# STATUS 2026-07-07

## 1. 本轮完成了什么

- 新增 `run_weighted_awrcore_models.py`，在保留 M0 `stable_plus mean-z` baseline 的基础上，构建了 M1/M2/M3 三类 AWR 连续状态解释层：
  - M1：source-only direction-corrected mean-z AWR。
  - M2：direction-corrected + effect/stability/redundancy weighted AWR。
  - M3：channel-constrained weighted AWR，输出 Hx、Hy、M3_equal、M3_weighted。
- 完成 Exp1 -> Exp2 与 Exp2 -> Exp1 双向外测协议：
  - 方向、权重和 high-AWR 状态阈值只由 source dataset 确定。
  - target dataset 只用于评价和图上标注。
  - Stage5 仅作为 late-state proxy label，不作为真实失效标签。
- 复用 BD v2，并基于 selected AWR 重新组合 RS、TES、AWR-BD state region 和 occupancy。
- 新增 `run_awrcore_fair_ablation.py`，完成公平消融：
  - stable_plus-only：比较 M0_stable / M1_stable / M2_stable。
  - no-corrdist：评估去掉 `rs_corrdist_base`、`rx_corrdist_base` 后的变化。
  - channel/family：比较 rx_only、ry_only、rs_only、rx_ry、rx_ry_rs、stable_plus。
- 当前结论：
  - stable_plus 内部最佳为 `M0_stable`。
  - `M1_stable` 与 `M0_stable` 基本一致，说明 stable_plus 特征方向冲突不明显。
  - `M2_stable` 未带来稳健增益，加权可能放大 source-specific 差异。
  - corrdist dependency 判定为 weak，corrdist 是贡献因素但不是 M0 优势的唯一来源。
  - 通道/族群消融显示 `ry_only_direction_mean_z` 排序能力突出，但推荐结构仍是 stable_plus 主分数 + 通道/特征族辅助解释。
- 已将本轮脚本和结果提交并推送到 GitHub：
  - commit: `9443e86 Add weighted AWR-core ablation outputs`
  - remote: `https://github.com/Rfox606/my_chatgpt_demo.git`

## 3. 下一步建议做什么

- 保持研究目标不变：继续把 AWR-BD-RS-TES 作为敏感相位力信号的连续状态表征层，不转向 Stage1-Stage5 分类或失效预警。
- 将最终 AWR 主结构暂定为：
  - `M0_stable` 作为主 AWR score。
  - direction sign、feature weight、Hx/Hy、channel/family ablation 作为解释层和透明性补充。
- 针对物理闭环验证优先检查以下区间和证据：
  - `M0_stable` 高且 `BDall_xy_v2` 高的窗口区间。
  - `ry_only_direction_mean_z` 与 stable_plus 轨迹差异较大的窗口区间。
  - `rs_corrdist_base`、`rx_corrdist_base` 贡献突出的窗口区间。
  - TES high-confidence event 附近的阶段边界和力信号形态变化。
- 建议下一轮生成一份面向物理闭环验证的候选窗口清单：
  - 每个 dataset 选出 high_AWR_high_BD、high_AWR_low_BD、low_AWR_high_BD 的代表窗口。
  - 对每个窗口列出 center_cycle、AWR、BD、RS、TES、top feature contribution 和对应图件路径。
- 在论文/报告表达上继续强调：
  - Stage5 是 late-state proxy label。
  - high AWR 是状态区划分阈值，不是失效预警阈值。
  - RS/TES 是依赖 AWR 的衍生状态指标，不能单独证明 AWR 的物理意义。
  - FEM、接触形貌、磨屑或实验观测仍是后续外部验证依据。
