# Codex任务书：部分共享动态原语与实验特异连续进程模型 v3

## 提交前固定的研究目标

本研究只检验一条主线：在不读取标签或未来样本的条件下，使用两个实验共有的力信号窗口特征学习**共享因果预测表示**与**动态原语先验**；随后由每个实验自己的过去数据独立决定状态数量、局部状态语义和可返回的在线状态路径；并在状态模型之外，独立给出实验条件化的连续进程分数。

共享层的训练目标是给定时刻 `t` 之前 32 个窗口的特征与差分，预测 `t+1` 的标准化特征（Huber 损失）。它不是时间排序器，任何全程时间排序、绝对磨损量或 Stage 均不进入损失或模型选择。动态原语是对共享预测表示中的因果动态描述符训练的 2--6 个 pooled Gaussian 原语；它们只以 posterior/prior 特征参与各实验的本地状态模型初始化与正则，不提供或复制目标状态中心。

每个实验独立在 `K={2,3,4,5,6}` 上按 BIC 选择其状态数量，并仅以该实验的因果描述符拟合局部状态中心、方差、转移和状态名称（`Exp1_LOCAL_*` / `Exp2_LOCAL_*`）。相同的偶然 K 值不视为对齐；不计算跨实验 state-ID 映射。状态允许返回、跳跃和 UNKNOWN，绝不施加单调约束。

连续进程分数不读取 state label/probability/centre/transition。它直接由共享预测器的因果残差、因果活动幅度和本实验滚动鲁棒分位标定构成；它是一个实验条件化的实数证据分数，不是分类、全程 rank 或绝对磨损量。

## 术语边界

- **共享**：仅共享输入特征定义、因果预测器参数和动态原语字典；不共享实验状态中心、K、state-ID、状态语义或状态路径。
- **动态原语**：在 pooled causal descriptor 上拟合的、无标签的局部动力学模式；不是通用退化阶段，也不等同于状态。
- **实验特异状态**：只由一个实验自身的历史描述符产生的离散在线隐变量，输出可回访且名称带实验前缀。
- **连续进程分数**：独立于状态模型的、条件化且因果的动态进程证据；不解释为磨损量或阶段。
- **未来**：评分时刻之后的任何窗口、全样本双向平滑、事后重拟合均视为未来数据。

## 非目标和禁止偏移

本版本不比较绝对磨损，不恢复跨实验五阶段分类，不将时间当作主要优化目标，也不把变化点检测当作唯一输出。下列行为禁止：

1. 强制两个实验使用相同 K；
2. 对齐、重编号或语义配对不同实验的 state-ID；
3. 将源实验 state centre/variance/transition 直接用于目标；
4. 使用全程时间排序作为训练、选择或验收主目标；
5. 输出固定五分类；
6. 只输出变化点而无独立连续进程分数；
7. 使用 `Stage`、形貌、磨屑、绝对磨损量或未来目标数据选择参数。

## 运行前锁定的配置与接受标准

输入固定为 `outputs_continuous_state_v45/results/window_feature_raw_v45.csv` 的 `F_core_v45` 九个力信号特征。因果上下文 32 windows，预测 horizon=1，预测表示维数=6，Huber `delta=1.35`，ridge `alpha=0.20`；动态原语 K candidates=`[2,3,4,5,6]`，bootstrap=30，最小 primitive 占比=3%；状态 K candidates=`[2,3,4,5,6]`，最小状态占比=3%，sticky=0.85，最短 dwell=8 windows；连续分数使用 residual/activity weights=`[0.65,0.35]`、滚动校准窗口=160、最低有效历史=32；prefix 审计 cutoff=`[0.35,0.60]`；随机种子 20260722。

在任何主模型运行之前，以下门槛固定：

| 项目 | 通过条件 |
|---|---|
| 因果预测 | 两实验各自的 held-out 后 30% 上，pooled shared predictor 的 1-step Huber MAE 不高于 persistence baseline 的 1.00 倍；同时 pooled 相对 MAE 改善平均值至少 1%。 |
| 原语稳健性 | 30 个 block bootstrap 中，每个实验对 pooled primitive assignment 的 adjusted Rand index 中位数至少 0.50，且有效 primitive 数至少 2。 |
| 实验特异状态 | 每实验独立 BIC 选 K（不得从另一个实验拷贝），所有 centre provenance=`local_experiment_only`，无跨实验 ID 映射；输出覆盖率 100%，且在合成回访轨迹上能返回先前状态。 |
| 连续进程输出 | 每实验 100% 有限输出，标准差大于 `1e-6`，不得读取状态输出；与原始输入的 prefix 比较必须最大绝对差 `<=1e-12`。 |
| 因果和标签审计 | 所有共享表示、状态、连续分数在两个 cutoff 的 prefix 差 `<=1e-12`；Stage/形貌/磨屑/绝对磨损/未来列读入次数为零。 |
| 工程验收 | 全部 pytest 通过，运行配置、输入 SHA256、每项门槛及成功/失败理由写入不可覆盖的结果目录和报告。 |

科学结论仅在上述所有门槛都通过时为 PASS；任何失败必须保留原始结果、失败原因、配置和报告，禁止据此事后调参重跑。
